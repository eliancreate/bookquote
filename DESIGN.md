# Passage 拾句 Design System

本文件是介面設計的單一依據。新畫面應沿用現有 React Native 元件與 token；除非現有做法無法滿足需求，不新增 UI 套件。

## 1. 視覺氣質

安靜、明亮、帶紙張感的閱讀工具。密度低至中等，留白優先，視覺重點是文字本身。介面不追求裝飾性；圓角、柔和灰階、低飽和藍灰色與克制的表面深度只用來建立層級。

- 密度：4/10，日常使用但保有呼吸感
- 變化：3/10，穩定、對稱、易讀
- 動態：3/10，克制且有觸覺回饋
- 主題：三套明亮主題與一套「午夜霓虹」深色主題

## 2. 主題與語意色

使用者可在設定頁即時切換主題，選擇保存於 `linekeep_theme`。每套主題的 `bg / panel / nav / ink / muted / accent / danger / line` 八個語意色可透過 HSV 連續色域與 HEX 色號獨立調整，保存於 `linekeep_palettes`；元件不得直接寫入主題色。

### 極簡色調（預設）

- `bg #F3F4F2`、`panel #FAFAF8`、`ink #252725`
- `muted #767B78`、`accent #718E9E`、`danger #C55252`、`line #DFE1DE`

### 花園藤椅

沉靜綠色與石材大地色，採用 Shell `#F7F5F1`、Stone `#E0DCD1`、Light Olive `#B4BD62`、Viridian `#344945`。

- `bg #F7F5F1`、`panel #E0DCD1`、`ink #344945`
- `muted #667264`、`accent #B4BD62`、`danger #A45244`、`line #C9C5B8`

### 午夜霓虹

唯一深色主題，背景接近黑夜但保留午夜藍色相。由 Midnight Blue `#364C84`、Lavender Mist `#D0D9F5`、Lemonade Green `#E7F1AB` 向更深、更高對比的功能色延伸；霓虹只作單一功能強調，不使用外發光。

- `bg #080D1C`、`panel #111A35`、`ink #F7F5F1`
- `muted #B7C3E7`、`accent #DDF57A`、`danger #F07AD9`、`line #24345F`
- 預設狀態列使用亮色圖示；自訂背景後依背景明度自動切換圖示

### 日光閃爍

藍天、白雲與日光黃，採用 Sky `#D5E3E8`、Sky Blue `#95B1EE`、Ivory Glow `#FFFDF5`、Naples `#FAD564`。

- `bg #D5E3E8`、`panel #FFFDF5`、`ink #1B475D`
- `muted #567482`、`accent #FAD564`、`danger #E86E49`、`line #95B1EE`

預設主題禁止背景漸層、純黑與彩色陰影；自訂色可接受完整 HEX 色域。每套介面配色各自保存「＋ 新增書摘」與「抽書籤」的獨立三色漸層，資料保存於 `passage_action_gradients`；切換主題時同步套用對應漸層。主題色不取代封面色；新書籍封面使用系統預設色彩。

## 3. 字體與層級

- **介面文字**：平台原生繁中字體；首頁 `Passage` 品牌標題使用 iOS `Baskerville`／Android serif 優雅斜體與 `600` 字重，其餘標題使用 `800`，一般標籤使用 `600–700`
- **書摘文字**：iOS 使用 `Songti TC`，Android 使用系統 serif
- **頁面標題**：21px / `800`，行高約 1.25
- **書摘**：18px / `500`，行高約 1.6，水平置中
- **書名**：14px / `700`；作者 13px / regular
- **輔助標籤**：最小 12px，可使用 1–3px 字距
- 介面任何文字不得小於 12px，正文與輸入文字不得小於 16px
- 內文單行寬度不要超過約 32 個中文字；不可為了塞入內容縮小字體

## 4. 間距與形狀

以 4px 為基準：`4 / 8 / 12 / 16 / 20 / 24 / 32`。

- 畫面左右留白：手機 20px；平板內容應置中並限制閱讀寬度
- 書摘卡片：圓角 22px、內距 20px、最小高度 190px；內容增加時自然增高
- 搜尋欄：最小高度 48px、圓角 16px
- 一般按鈕：最小高度 46px、圓角 14px
- 圖示按鈕與所有觸控目標：至少 44×44px
- Bottom sheet：頂部圓角 30px，一般最大高度 92%；新增／編輯書摘使用 safe area 內 96% 高度
- Bottom sheet 顯示時背景保持原色，不加暗色遮罩或模糊效果
- 首頁書摘卡與書櫃書本保留細邊與柔和陰影；其餘卡片與按鈕不使用邊線，只保留極淡表面陰影。輸入框與選色游標可保留操作性邊界

## 5. 核心元件

### 書摘卡片

- 使用 `Warm Card`，搭配低對比細邊與極淡同色系陰影
- 書摘內容在可用區域水平與垂直置中，不顯示外層引號
- 書名與作者靠齊右下角
- 預設只顯示書摘、書名、作者
- 向左滑動最多 112px，才露出編輯與刪除操作；一般狀態不可看見操作按鈕
- 滑動以 `transform` 完成，不改變卡片寬度或列表排版

### 按鈕

- 主按鈕：Charcoal Ink 底、白字
- 次按鈕：Warm Card 底，不使用邊線
- 危險按鈕：淡紅底與 Danger 文字
- 按下時縮放至 `0.97` 並略降透明度，形成實體按壓感；不可使用外發光

### 輸入與彈窗

- 標籤在輸入框上方，錯誤就地顯示或使用原生 Alert
- 輸入框最小高度 48px；多行輸入最小高度 120px
- Bottom sheet 必須避開鍵盤與四邊 safe area；新增／編輯書摘在 iOS 鍵盤開啟時縮放可用高度，不整張向上位移

### 底部導航與 FAB

- 導航列為避開底部 safe area 的浮動不透明膠囊，使用每套主題可獨立設定的 `nav` 色；不使用邊線，底部陰影保持窄而淡，固定使用深色
- 內容向下捲動（手指上滑）時導航向下隱藏；內容向上捲動（手指下滑）時重新出現，並以位移門檻避免抖動
- 導航隱藏動畫只使用 `transform` 與 `opacity`，並尊重系統減少動態設定
- 導航圖示置於獨立不透明前景層，不套用模糊、光暈或文字陰影
- FAB 為 58×58px，使用約 70% 不透明的獨立三色漸層、低強度磨砂與跟隨漸層中間色的極淡光暈，必須位於導航列與系統 safe area 上方
- 目前分頁使用 `accent` 實心圖示，未選取分頁使用 `muted` 輪廓圖示；不增加膠囊底或文字標籤

### 搜尋與書籍篩選

- 頂部品牌只顯示「拾句」；首頁右上放置最小高度 44px 的長條「抽一張」按鈕
- 搜尋與篩選位於品牌列下方；首頁不顯示固定搜尋欄，點擊放大鏡後才橫向展開
- 搜尋欄展開時保留至少 44px 的關閉按鈕，且不得擠壓至不可輸入
- 篩選使用 `filter-variant` 圖示與 bottom sheet，可複選一本或多本書
- 已套用篩選時在圖示上顯示所選數量；清除全部代表顯示所有書摘

### 書櫃展示

- 頁面不顯示「書櫃展示」標題，頂部直接提供「書背展示／書封展示」切換
- 書封展示使用手機雙欄；書背展示使用單欄橫向書背，長書名可換行並增高
- 切換控制至少 44px 高，狀態必須同時以底色與文字表達

### 首頁抽卡

- 首頁右上長條按鈕使用獨立三色漸層與跟隨漸層中間色的極淡光暈；當天尚未抽取時偶爾播放一次掃光，抽取日期保存於 `passage_last_draw_date`，抽過後當天停止
- 抽取結果顯示於列表最上方，使用淺藍灰卡片、置中書摘與右下書名作者
- 沒有可用書摘時按鈕為 disabled；長文字讓結果卡自然增高

## 6. 響應式與安全區域

- 以手機直向為基準；iPhone、Android 手勢導航與平板都必須使用 safe-area inset
- 浮動按鈕、底部導航、Modal 與鍵盤不可互相遮擋
- 小螢幕維持單欄；只有書櫃封面可使用雙欄
- 多欄內容若不足以保留 44px 觸控目標與 16px 正文，改為單欄
- 長文字必須換行並使容器增高，不截斷書摘
- 禁止水平溢出；平板不應無限制拉寬閱讀文字

## 7. 動態與互動

- 動畫只使用 `transform` 與 `opacity`
- 點擊回饋約 150–250ms；滑動回位使用無彈跳 spring
- 尊重系統減少動態設定；功能不能依賴動畫才能理解
- 每個圖示按鈕必須有 `accessibilityLabel`，破壞性操作必須再次確認

## 8. 禁止模式

- 不使用純黑、紫藍背景漸層或多重頁面強調色；多色漸層與極淡外光暈只限兩個主要操作
- 不在卡片與導航加裝飾性重邊或高對比通用陰影；深度必須沿用一致的表面 token
- 不使用巢狀卡片、三欄等寬功能卡或固定高度截斷長文字
- 不使用 emoji 取代圖示，不混用不同筆畫風格的圖示
- 不加入無用途的持續動畫、全頁玻璃擬態、噪點或背景照片；玻璃效果僅限浮動導航，掃光僅限當天尚未抽取的提示
- 不使用「無縫、顛覆、釋放潛能」等空泛文案
