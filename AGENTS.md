# 拾句 App UI 工程規則

本檔案適用於整個專案。每次修改 UI、互動或版面前後都必須遵守；不得以「只改一點」為理由略過。

## 修改前

- 先閱讀 `DESIGN.md`；涉及觸覺回饋時再閱讀 `HAPTICS.md`。
- 確認變更會出現在哪些狀態：一般、空白、長文字、鍵盤開啟、Modal、載入與錯誤。
- 優先使用 React Native、Expo 與既有依賴；不要為簡單版面新增套件。

## Safe area（強制）

- 所有全螢幕頁面、Modal、底部導航、FAB、固定按鈕與 bottom sheet 都必須避開上下左右 safe-area inset。
- 原生 `Modal` 必須在 Modal 內容內建立 `SafeAreaProvider`，再使用 `SafeAreaView` 或 `useSafeAreaInsets`；不可假設根層 Provider 的座標可跨越 Modal。
- 頂部返回、關閉、標題不可只靠固定 `paddingTop` 避開狀態列。
- 底部導航高度必須包含 `insets.bottom`；FAB 的 bottom 必須包含底部導航與 `insets.bottom`。
- 鍵盤開啟時，輸入框、主要操作與關閉方式必須仍可觸及。
- 不用裝置型號、瀏海高度或魔術數字推測 safe area。

## 防止破版（強制）

- 以 320、375、430 與 768pt 寬度思考版面；不得只對單一手機尺寸調整。
- 文字容器使用 `minHeight` 而非固定高度；長書摘、長書名與系統放大字體必須能自然換行與增高。
- 互動目標至少 44×44pt；正文至少 16px，任何文字不得小於 12px。
- 不允許水平溢出、被截斷的主要內容、重疊控制項或被導航／鍵盤遮住的操作。
- 多欄在空間不足時必須降為單欄；不要用硬編碼百分比或 `calc` 類型的尺寸補丁。
- 絕對定位僅用於 FAB、滑動操作層等必要情境，並且必須把 safe area 納入位置計算。
- 動畫只改變 `transform` 與 `opacity`，不得讓動畫中的尺寸改變造成列表跳動。

## 互動與無障礙

- 每個圖示按鈕都要有明確 `accessibilityLabel`；不能只靠顏色或觸覺傳達狀態。
- 所有流程都要有返回或關閉路徑；破壞性操作必須確認。
- 手勢不能阻擋垂直捲動，且要提供可理解的替代操作或可收回狀態。
- 觸覺回饋只用於狀態改變、主要操作與成功／警告／錯誤，並同步更新 `HAPTICS.md`。

## 每次修改後的最低驗證

1. 執行 `npm run lint`。
2. 執行 `git diff --check`。
3. 檢查受影響頁面的短／長內容、空白狀態、Modal、鍵盤與返回流程。
4. 檢查頂部狀態列、底部 Home Indicator／Android 手勢列與左右安全區。
5. 確認 320pt 小螢幕與平板寬度不重疊、不截斷、不產生水平捲動。

未完成上述檢查，不得宣告 UI 修改完成。

## 備忘（強制）

- 主動跳過、延後或刻意簡化的工作，必須追加到 `MEMO.md`，並寫明重新評估的觸發條件。
- 已完成或確定不再需要時，從 `MEMO.md` 移除；不要讓備忘變成永久垃圾桶。
